# UA Parking System - Vercel Frontend Deployment Guide

## Overview
This guide prepares and deploys the frontend static HTML/CSS/JS application to Vercel, a platform optimized for frontend deployment with CDN, automatic HTTPS, and instant deployments.

---

## Prerequisites
- Active Vercel account (free tier available)
- GitHub repository with latest code pushed
- Render backend deployed and URL available
- Domain name (optional - Vercel provides free domain)

---

## Step 1: Prepare Frontend for Production

### 1.1 Update API Base URLs in All HTML Files

**Search & Replace Pattern**: Update in all 23 HTML files:

```javascript
// LOCAL DEVELOPMENT
const API_BASE_URL = 'http://localhost:8000/api';

// PRODUCTION (Render Backend)
const API_BASE_URL = 'https://ua-parking-backend.onrender.com/api';
```

**Files to Update:**
- index.html
- login.html, register.html, setup-profile.html, loading-page.html
- driver-*.html (5 files)
- guard-*.html (7 files)
- admin-*.html (6 files)

### 1.2 Create `vercel.json` Configuration

Create `vercel.json` in project root:

```json
{
  "buildCommand": "echo 'Frontend build complete'",
  "outputDirectory": "frontend",
  "public": false,
  "env": {},
  "regions": ["iad1"],
  "functions": {
    "api/**": {
      "memory": 1024,
      "maxDuration": 60
    }
  },
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/$1"
    },
    {
      "source": "/:path((?!.*\\..*).*)",
      "destination": "/index.html"
    }
  ],
  "headers": [
    {
      "source": "/api/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=3600"
        }
      ]
    },
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "X-Content-Type-Options",
          "value": "nosniff"
        },
        {
          "key": "X-Frame-Options",
          "value": "SAMEORIGIN"
        }
      ]
    }
  ]
}
```

### 1.3 Create `.vercelignore` File

Create `.vercelignore` in project root:

```
backend/
.git
.venv
node_modules
*.py
*.pyc
__pycache__
*.env
requirements.txt
manage.py
db.sqlite3
media/
staticfiles/
.gitignore
README.md
```

### 1.4 Optimize Assets (Optional)

For better performance:
- Minimize HTML, CSS, JavaScript
- Compress images (use tools like TinyPNG)
- Use WebP format for images where possible
- Lazy load images where applicable

### 1.5 Create Production Configuration File

Create `frontend/.env.production.local`:

```env
VITE_API_URL=https://ua-parking-backend.onrender.com/api
VITE_APP_NAME=UA Parking System
VITE_APP_VERSION=1.0.0
```

## Step 2: Deploy to Vercel

### 2.1 Connect GitHub Repository

1. Go to [vercel.com](https://vercel.com)
2. Click "Add New..." → "Project"
3. Import GitHub repository
4. Select `grouptatlo` repository
5. Authorize Vercel with GitHub

### 2.2 Configure Project Settings

| Setting | Value |
|---------|-------|
| **Project Name** | `ua-parking-system` |
| **Framework** | Other (static HTML) |
| **Root Directory** | `frontend` |
| **Build Command** | `echo 'build complete'` (no build needed for static files) |
| **Output Directory** | `.` (current directory) |

### 2.3 Set Environment Variables

In Vercel Dashboard → Settings → Environment Variables:

```
VITE_API_URL=https://ua-parking-backend.onrender.com/api
VITE_APP_NAME=UA Parking System
```

### 2.4 Deploy

1. Click "Deploy"
2. Vercel builds and deploys automatically
3. Get deployment URL: `https://ua-parking-system-xyz.vercel.app`
4. Automatic URL also provided: `ua-parking-system.vercel.app`

### 2.5 Configure Custom Domain (Optional)

1. Go to Settings → Domains
2. Add custom domain (e.g., `parking.ua.edu.ph`)
3. Follow DNS configuration steps
4. SSL certificate auto-generated by Let's Encrypt

## Step 3: Update Backend for Frontend Domain

### 3.1 Update CORS Settings in Render

Go to Render Backend → Environment Variables:

```env
CORS_ALLOWED_ORIGINS=https://ua-parking-system.vercel.app,https://parking.ua.edu.ph,https://ua-parking-backend.onrender.com
```

### 3.2 Restart Backend Service

In Render Dashboard:
- Click "Manual Deploy" or wait for auto-redeploy
- Verify API is accessible from frontend

## Step 5: Advanced Configuration

### 5.1 Caching Strategy

Add to `vercel.json`:

```json
"headers": [
  {
    "source": "/(css|js|images)/(.*)",
    "headers": [
      {
        "key": "Cache-Control",
        "value": "public, max-age=31536000, immutable"
      }
    ]
  },
  {
    "source": "/*.html",
    "headers": [
      {
        "key": "Cache-Control",
        "value": "public, max-age=3600, must-revalidate"
      }
    ]
  }
]
```

## Common Issues & Solutions

### Issue: CORS Error - "Access to XMLHttpRequest blocked"
**Solution**: 
- Verify frontend URL added to backend `CORS_ALLOWED_ORIGINS`
- Check API URL in HTML files matches deployed backend
- Wait 2-3 minutes for env var changes to propagate

### Issue: "Failed to fetch" on API calls
**Solution**:
- Verify backend is running: `curl https://ua-parking-backend.onrender.com/api/`
- Check network tab in browser console
- Verify frontend API URL is correct
- Check firewall/CORS headers

### Issue: QR Codes not displaying
**Solution**:
- Verify media files served from backend
- Check QR code file path: `https://ua-parking-backend.onrender.com/media/qr_codes/...`
- Test image URL directly in browser
- Verify backend static/media files configured

### Issue: Login page doesn't load
**Solution**:
- Check frontend URL: `https://ua-parking-system.vercel.app/`
- Verify index.html loads (should see loading animation)
- Check logo.png is in frontend directory
- Clear browser cache

### Issue: Session data lost after refresh
**Solution**:
- Verify LocalStorage is working
- Check browser privacy mode (disables LocalStorage)
- Test in normal mode, not incognito
- Check browser console for errors

## Performance Tips

### Frontend Optimization
1. **Lazy load images** - Load on demand
2. **Compress assets** - Use gzip compression
3. **Minimize CSS/JS** - Remove unused code
4. **Cache strategy** - Cache static files 1 year
5. **CDN** - Vercel uses global CDN by default

### Expected Performance Metrics
- **First Contentful Paint (FCP)**: < 1.5s
- **Largest Contentful Paint (LCP)**: < 2.5s
- **Cumulative Layout Shift (CLS)**: < 0.1

## Rollback Procedure

### If Frontend Has Issues

**Vercel**:
1. Go to Deployments tab
2. Find previous working deployment
3. Click "Promote to Production"
4. Automatic rollback in < 1 minute

### If Backend Has Issues

**Render**:
1. Go to Web Service → Deployments
2. Select previous working build
3. Click "Deploy"
4. Service restarts with old code
